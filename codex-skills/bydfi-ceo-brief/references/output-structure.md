# Output Structure

Use this as the default structure for a management-facing BYDFI CEO brief.

## Default Section Order

1. `一句话判断`
2. `今天必须拍板的三件事`
3. `会议级待办追踪页`
4. Top problem lines, ordered by business severity
5. `Ella 项目进度表与周报串联` when perpetual / Ella / cross-role project chaining matters
6. `解决方案落地` as a mandatory management-closing section
7. Supporting lines such as translation coordination or efficiency systems
8. `关键执行人信号`
9. `暂不建议下判断的事项`

## Meeting Tracker

The meeting tracker is not optional when the user asks for post-meeting follow-up.

Required table:

`会议 / 待办进展 / 结果缺口 / 今日新增待办`

Only keep meetings that matter to management. Do not dump every meeting title.

## Ella Project Table

Use the Ella table when product, frontend, backend, and testing statuses can be chained around the same item.

Required table:

`事项 / 当前阶段 / 跨角色串联 / 主责任人 / 下一里程碑`

The table should answer one management question: which items are truly closed, which are near close, and which are still cross-role parallel noise.

## Solution Landing

Every CEO-facing PDF should end with a management action page, not just a diagnosis page.

Required framing:

`问题 / 解法 / 负责人 / 时点 / 验收标准`

The purpose of this section is to reduce decision burden for the CEO. Do not stop at "what happened" or "what is wrong". Close the report by stating:

- what must be changed now
- who owns the change
- when management should expect a result
- how success will be judged

Hard rule:

- No solution item without an owner.
- No solution item without a time checkpoint.
- No solution item without an acceptance metric.
- Use result metrics, not process narration, as the acceptance standard.

## Writing Standard

- Lead with judgment, not evidence logs.
- Prefer business consequence over implementation chronology.
- Use recency-weighted evidence.
- Keep direct source residue out of the management brief.
- If a fact is thin, downgrade the confidence instead of padding with rhetoric.
- Final tone should read like a senior operator or analyst writing for management, not a consultant presenting a framework.
